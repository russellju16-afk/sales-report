# 2025-12-24 v1.1 state/ux patch

## 变更摘要
- title/subtitle 动态化
- URL + localStorage 状态持久化
- order-link 按钮化与事件委托
- 表格 sort 指示
- filter-row sticky
- 清空表头筛选
- 新增 state.js
- README 自测清单

## 为何做
- 让页面标题、数据范围与版本信息与数据一致，减少人工改文案
- 让分段/时间筛选等状态可被分享与恢复，降低重复操作
- 统一订单入口的交互样式并优化事件绑定效率
- 提升表格排序、筛选的可见性与可用性

## 收益
- 业务数据与页面信息同步，降低误读风险
- 支持链接分享与刷新后恢复状态
- 更清晰的排序/筛选反馈，减少误操作
- 事件委托减少节点监听，降低性能开销

## 潜在回归点
- URL 与 localStorage 状态恢复可能覆盖默认初始状态
- order-link 样式替换为按钮后，CSS 或点击区域可能变化
- 表头排序指示与 sticky 叠加时可能影响布局
- 清空表头筛选可能影响已有筛选流程

## 自测项
- 刷新页面后：分段、时间范围、表内筛选状态是否可恢复
- 复制 URL 分享后：打开是否保持相同视图与筛选
- 订单入口按钮点击是否仍能正确打开弹窗与复制
- 表格排序时：指示符是否正确更新
- 表头筛选清空按钮是否可用且不影响其它筛选
- filter-row sticky 时：滚动对齐与遮挡是否正常
